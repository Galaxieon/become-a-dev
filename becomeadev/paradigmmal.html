PROGRAMMING PARAGIGM എന്നത് പ്രോഗ്രാമിംഗിന്റെ ശൈലിയെയോ രീതിയെയോ സൂചിപ്പിക്കുന്നു.. പ്രോഗ്രാമിംഗിലെ വിവിധ സമീപനങ്ങൾ ഇവയാണ് modular, top-down, bottom–up and structured programming.

C++ implements two types of Paradigms, procedural and object oriented paradigm.

Procedure oriented programming paradigm

Procedure oriented programming ഒരു കൂട്ടം നിർദ്ദേശങ്ങൾ ഉൾക്കൊള്ളുകയും ഈ നിർദ്ദേശങ്ങൾ ഫംഗ്ഷനുകളായി ക്രമീകരിക്കുകയും ചെയ്യുന്നു.Programming using high-level languages such C, COBOL is known as procedure-oriented programming.

Limitations of Procedure oriented programming are,

  1. Data is undervalued

    2. പ്രൊസീഡ്യൂറൽ പ്രോഗ്രാമിംഗ് കാര്യങ്ങൾ ചെയ്യുന്നതിന് പ്രാധാന്യം നൽകുന്നു. ഡാറ്റയ്ക്ക് പ്രാധാന്യം കുറവാണ്, ഏത് ഫക്ഷനും ഡാറ്റ ആക്സസ് ചെയ്യാനും മാറ്റാനും കഴിയും.

  2. Adding new data element needs modification to functions
  3. Difficult to create new data types

    4. പുതിയ data type സൃഷ്ടിക്കാനുള്ള പ്രോഗ്രാമിംഗ് ഭാഷയുടെ കഴിവിനെ ആണ് Extensibility.

  4. Poor real world modeling

Object Oriented Programming Paradigm

ഒബ്ജക്റ്റ് ഓറിയന്റഡ് പ്രോഗ്രാമിംഗ് ഡാറ്റയും ഫംഗ്ഷനും ഒബ്ജക്റ്റ് എന്ന ഒരൊറ്റ യൂണിറ്റിലേക്ക് ബന്ധിപ്പിക്കുന്നു..Objects communicate with each other by using functions.

Advantages of Object Oriented Programming

  • Easy to maintain and modify code.
  • Provides modular structure for programs.
  • Code sharing.
  • Information hiding.
  • It is good for defining abstract data type.
  • It implements real life scenario.
