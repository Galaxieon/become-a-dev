പോയിന്ററും സ്ട്രിങ്ങും (pointer and string)

സ്ട്രിങ്ങനെ കാരക്ടർ അറെ വഴി ശേഖരിക്കാമെന്നും അറെയുടെ പേരിനെ സ്ട്രിങ് വേരിയബിളായി കണക്കാക്കാമെന്നും നമ്മൾ 11-ന്നാം ക്ലാസ്സിൽ പഠിച്ചു. അറെയുടെ പേരിൽ അതിന്റെ ബേസ് അഡ്രസ്സ് അടങ്ങിയിരിക്കുന്നു എന്നും നമ്മൾ മനസ്സിലാക്കി. അതുകൊണ്ട് അറെയെ ഒരു പോയിന്റ്റായി കണക്കാക്കാം. ഇനി നമുക്ക് ഈ ആശയങ്ങളെ യോജിപ്പിച്ചുകൊണ്ട് ഒരു സ്ട്രിങ്ങിനെ പോയിന്ററിലൂടെ എങ്ങനെ ഉപയോഗിക്കാമെന്നും അതിന് മറ്റ് പോയിന്റ്ററുമായി എന്ത് വ്യത്യാസമുണ്ടെന്നും ചർച്ച ചെയ്യാം.

താഴെ കൊടുത്ത വാചകങ്ങൾ പരിശോധിക്കുക.

char str[20]; / / ക്യാരക്ടർ അറെ പ്രഖ്യാപനം
char *sp; / / ക്യാരക്ടർ പോയിന്റർ പ്രഖ്യാപനം
cin>>str; / /"program" എന്ന സ്ട്രിങ്ങിനെ ഇൻപുട്ട് ചെയ്യുന്നു എന്ന് കരുതുക
cout<<str; / /"program" എന്ന സ്ട്രിങ്ങിനെ പ്രദർശിപ്പിക്കാൻ
sp=str; / /str എന്ന സ്ട്രിങ്ങിന്റെ ഉള്ളടക്കം, പോയിന്റർ sp യിലേക്ക് കോപ്പി ചെയ്യുന്നു.
cout<<sp; / /"program" എന്ന സ്ട്രിങ് പ്രദർശിപ്പിക്കുന്നു
cout<<&str[0]; / /"program" എന്ന സ്ട്രിങ് പ്രദർശിപ്പിക്കുന്നു
cout<<sp+1; / /"program" എന്ന സ്ട്രിങ് പ്രദർശിപ്പിക്കുന്നു
cout<<&str+1; / /"program" എന്ന സ്ട്രിങ് പ്രദർശിപ്പിക്കുന്നു
cout<<str[0]; / /'p' എന്ന ക്യാരക്ടർ പ്രദർശിപ്പിക്കുന്നു
cout<<*sp; / /'p' എന്ന ക്യാരക്ടർ പ്രദർശിപ്പിക്കുന്നു
cout<<*&str; / /str അറെയുടെ ബേസ് അഡ്രെസ്സ് പ്രദർശിപ്പിക്കുന്നു
cout<<&sp; / /sp എന്ന പോയിന്ററിന്റെ വിലാസം പ്രദർശിപ്പിക്കുന്നു

കാരക്ടർ പോയിന്ററിന്റെ നേട്ടങ്ങൾ

സ്ട്രിങ്ങിനെ സ്റ്റോർ ചെയ്യാൻ കാരക്ടർ പോയിന്റർ ഉപയോഗികുമ്പോൾ ചുവടെ കൊടുത്ത നേട്ടങ്ങൾ ഉണ്ടാകുന്നു.

  • വലുപ്പം പ്രഖ്യാപന പ്രസ്താവനയിൽ രേഖപ്പെടുത്തതുകൊണ്ട്, എത്ര വലുപ്പമുള്ള സ്ട്രിങ്ങും ശേഖരിക്കാം. മെമ്മറി പാഴാക്കപ്പെടുകയോ തികയാതിരിക്കുകയോ ചെയ്യുന്നില്ല. എന്നാൽ ഇത് പ്രാരംഭവില നൽകുമ്പോൾത്തന്നെ ചെയ്തിരിക്കണം. (eg, char *str = "program";)
  • വില നൽകാൻ ഓപ്പറേറ്റർ (=) ഉപയോഗിച്ച് സ്ട്രിങ് കോപ്പി ചെയ്യാം.
  • സ്ട്രിങ്ങിലെ ഏത് കാരക്ടറും പോയിന്റർ ഉപയോഗിച്ച് എളുപ്പം പുറത്തെടുക്കാം.
  • കാരക്ടർ അറകളെ കറുത്ത മെമ്മറി സ്ഥലം ഉപയോഗിച്ച് കൈകാര്യം ചെയ്യാം.
