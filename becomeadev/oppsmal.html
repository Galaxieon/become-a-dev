The main concepts of Object Oriented Programming are

  1. Objects: - ഒബ്ജക്റ്റുകൾ ഒരു വ്യക്തി, വിദ്യാർത്ഥി, ബുക്ക് കാർ, ടിവി ... മുതലായ യഥാർത്ഥ ലോക വസ്തുക്കളാണ്. ഒബ്ജക്റ്റുകൾ ഡാറ്റയുടെയും പ്രവർത്തനങ്ങളുടെയും സംയോജനമാണ് സന്ദേശങ്ങൾ അയച്ചുകൊണ്ട് objects പരസ്പരം communicate ചെയ്യുന്നു
  2. Classes: - ഒബ്ജക്റ്റുകളുടെ ഒരു ശേഖരമാണ് ക്ലാസ്.An object is an instance of a class.
  3. Abstraction: - Abstraction refers to മറ്റ് വിശദാംശങ്ങൾ മറച്ചുവെച്ച് അവശ്യ സവിശേഷതകൾ പ്രദർശിപ്പിക്കുന്നു. .അബ്സ്ട്രാക്ഷൻ ഡാറ്റ സുരക്ഷ നൽകുന്നു.
  4. Encapsulation: - ഡാറ്റയും പ്രവർത്തനവും ഒരൊറ്റ യൂണിറ്റിലേക്ക് പൊതിയുന്നതിനെ എൻക്യാപ്സുലേഷൻ എന്ന് വിളിക്കുന്നു
  5. Modularity: - ഒരു വലിയ പ്രോഗ്രാമിനെ മൊഡ്യൂളുകൾ എന്ന് വിളിക്കുന്ന ചെറിയ പ്രോഗ്രാമുകളായി വിഭജിക്കുന്ന പ്രക്രിയയാണ് മോഡുലാരിറ്റി.
  6. 6) Inheritance: - ഒരു ക്ലാസിലെ ഒബ്ജക്റ്റുകൾ മറ്റൊരു ക്ലാസിലെ വസ്തുക്കളുടെ സവിശേഷതകൾ നേടിയെടുക്കുന്ന പ്രക്രിയയാണ് Inheritance. ഇൻഹെ-റിറ്റൻസ് എന്ന ആശയം പുനരുപയോഗം നൽകുന്നു

    7. The existing class is called base class and the new class is called derived class. The different forms of inheritance are Single inheritance, multiple inheritances, multilevel inheritance, hierarchical inheritance and Hybrid inheritance.

    ഒരു ആപ്പിളും വാഴപ്പഴവും പരിഗണിക്കുക. ഒരു ആപ്പിളും വാഴപ്പഴവും വ്യത്യസ്ത പഴങ്ങളാണെങ്കിലും,both are fruits.

  7. Polymorphism:-‘Poly’ means many, ’Morph’ means shape. ഒന്നിലധികം രൂപങ്ങൾ എടുക്കുന്നതിനുള്ള ഒബ്ജക്റ്റർ ഫംഗ്ഷന്റെ കഴിവാണ് പോളിമോർഫിസം

Types of polymorphism

  1. Compile time (Early binding / Static)polymorphism.
  2. Run-time (Late binging /Dynamic) polymorphism

Compile time polymorphism

ഫംഗ്ഷൻ ഓവർലോഡിംഗ്, ഓപ്പറേറ്റർ ഓവർലോഡിംഗ് എന്നിവയിലൂടെ കംപൈൽ ടൈം പോളിമോർഫിസം കൈവരിക്കുന്നു. ഫംഗ്ഷൻ ഓവർലോഡിംഗിൽ, ഫംഗ്ഷന് ഒരേ പേരുണ്ടെങ്കിലും വ്യത്യസ്ത ആർഗ്യുമെന്റുകൾ ഉണ്ട്.

Run-time polymorphism

വിർച്വൽ ഫംഗ്ഷനുകളിലൂടെയാണ് റൺ-ടൈം പോളിമോർഫിസം നേടുന്നത്. എക്സിക്യൂഷൻ (റൺ) സമയത്താണ് ഫംഗ്ഷൻ നിർവചനവുമായി ഒരു ഫംഗ്ഷനെ ബന്ധിപ്പിക്കുകയോ ലിങ്കുചെയ്യുകയോ ചെയ്യുന്നത്

Different types of Inheritance

Single Inheritance

When a derived class inherits properties and behaviors of only one base class, it is called single inheritance. Cricketer is a derived class from the base class Player.

Multiple inheritances

ഒരു derived ക്ലാസ് ഒന്നിൽ കൂടുതൽ അടിസ്ഥാന ക്ലാസിന്റെ ഗുണങ്ങളും പെരുമാറ്റങ്ങളും അവകാശമാക്കുമ്പോൾ അതിനെ multiple inheritance എന്ന് വിളിക്കുന്നു. In following figure, All Rounder is a derived class from two base classes: Bowler and Batsman.

Hierarchical Inheritance

ഒരു അടിസ്ഥാന ക്ലാസിന്റെ ഗുണങ്ങളും പെരുമാറ്റങ്ങളും ഒന്നിലധികം ക്ലാസ്സിൽ നിന്ന് പാരമ്പര്യമായി ലഭിക്കുമ്പോൾ, അതിനെ hierarchical inheritanceഎന്ന് വിളിക്കുന്നു. In following figure, Bowler and Batsman are two derived classes from same base class Cricketer.

Multilevel Inheritance

ക്ലാസിന്റെ ഗുണങ്ങളും രീതികളും മറ്റൊരു ക്ലാസ് base class ലഭിക്കുമ്പോൾ അതിനെ multi level എന്ന് വിളിക്കുന്നു

Hybrid Inheritance

It is combination of multiple and multilevel inheritance.
