Compilation സമയത്ത് ഒരു വേരിയബിളിനായി മെമ്മറി അനുവദിക്കുന്നത് സ്റ്റാറ്റിക് മെമ്മറി അലോക്കേഷൻ എന്നറിയപ്പെടുന്നു. compilation സമയത്ത് മെമ്മറി അനുവദിച്ചുകഴിഞ്ഞാൽ അത് expand or compress ചെയ്യാനോ കഴിയില്ല.

Example

int a[10] as integer;
During compilation the compiler will allocates 40 bytes

എക്സിക്യൂഷൻ സമയത്ത് മെമ്മറി അനുവദിക്കുന്ന പ്രക്രിയയെ ഡൈനാമിക് മെമ്മറി അലോക്കേഷൻ എന്ന് വിളിക്കുന്നു.This is done using ‘new’ and ‘delete’ operators.

Memork leak ഒരിക്കൽ ഒരു വേരിയബിളിന് മെമ്മറി ഡൈനാമിക് ആയി അനുവദിക്കുകയും പിന്നീട് അതിന്റെ ഉപയോഗത്തിന് ശേഷം മെമ്മറി ഡി-അലോക്കേറ്റ് ചെയ്യാതിരിക്കുകയും ചെയ്താൽ, മെമ്മറി നഷ്ടപ്പെടും. മെമ്മറി dynamic അനുവദിച്ചെങ്കിലും റിലീസ് ചെയ്യാത്ത ഈ പ്രശ്നമാണ് മെമ്മറി ലീക്ക്.

new and delete operators

execution സമയത്ത് (run-time) മെമ്മറി അനുവദിക്കുന്നതിന് new operator ഉപയോഗിക്കുന്നു.

The Syntax is

pointer_variable=new data_type;

new operator ഉപയോഗിച്ച് അനുവദിച്ച മെമ്മറി സ്വതന്ത്രമാക്കാൻ delete operator ഉപയോഗിക്കുന്നു

The syntax is

delete pointer_variable;
